## 0 · 项目一句话

一个微信**小程序**（不是小游戏）：把「记梦 → 织梦 → 显影 → 安放 → 归还 → 入睡」做成一场夜间陪伴仪式。
完整设计见 `verdant-mirage-product-design.html`（**权威文档，有冲突以它为准**）。

**当前阶段**：仅有设计文档 + 开屏动画，尚无代码。下一步 = Web 原型先行（`prototype/style-lab.html`），验证后再移植小程序。

## 1 · 角色分工（谁做哪一层）

| 层级 | 角色 | 由谁扮演 | 职责 | 何时出手 |
|---|---|---|---|---|
| T1 架构/评审 | **Architect** | Claude（Cowork，token 贵） | 定架构、写本宪法、拆任务、**评审安全层与关键代码**、裁决分歧 | 只在里程碑与高风险处，**不做日常编码** |
| T2 主力编码 | **Coder** | Claude Code + DeepSeek | 按任务单实现功能、写测试、跑原型 | 持续运行，承担绝大多数工作量 |
| T2 评审 | **Reviewer** | Claude Code + DeepSeek（**另开一个会话**） | 对 Coder 的产出做对抗式评审、找 bug、跑验收清单 | 每个任务单实现完就介入 |
| T3 素材 | **Artist** | GPT（图像，经 CC 调用） | 按「风格宪法」生成/精修静态素材 | 需要新美术素材时 |

**Token 经济学（重要）**：贵的脑子（Architect/Claude）不进运行时循环。它的思考沉淀成**仓库里的文件**（本宪法、路线图、任务单），让便宜的 agent 每次读文件即可，不必反复花贵 token。Architect 只在关键点评审。

## 2 · 协作协议：Coder ↔ Reviewer 自动对话

目标：**AI 之间自己交接，少由人传话。** 用 [`agmsg`](https://github.com/fujibee/agmsg)（bash + SQLite 的跨 agent 消息总线，装成 agent skill，无守护进程）打通两个会话。

**当前落地形态（只有 CC+DeepSeek 时）**：开**两个** Claude Code 会话，同一个 repo，一个 owner=`coder`、一个 owner=`reviewer`，都装 agmsg。它们通过共享 SQLite 互发消息，你只在旁边看。

标准循环：

1. `coder` 从 `ROADMAP.md` / `tasks/` 领一个任务单 → 在 `feature/T-xxx` 分支实现 → 自测。
2. `coder` 用 agmsg 发：`@reviewer T-xxx 完成，分支 feature/T-xxx，请按任务单的评审清单过一遍`。
3. `reviewer` 拉分支、对照任务单验收清单评审 → agmsg 回：通过 / 问题列表。
4. `coder` 修 → 复审。两方一致后：
   - **普通任务**：合并，标记任务单完成。
   - **触发升级规则（见 §5）的任务**：agmsg 标记 `NEEDS-ARCHITECT`，**停下等 Claude 评审**，不要自行合并。

**升级路径**：Coder/Reviewer 拿不准或触发红线时，把问题浓缩成一段写进 `tasks/ESCALATIONS.md`，人把它带给 Claude；Claude 的裁决写回同文件，成为新的共享上下文。

**分支/提交约定**：分支 `feature/T-xxx`；提交信息首行 `[T-xxx] 简述`；不直接推 `main`。

> 升级项（更强的模型多样性，日后再做）：装 OpenAI **Codex CLI**（你已有 OpenAI key），让 `reviewer` 换成 GPT——**跨厂商评审比同模型自审能多抓一类 bug**。或用 [`codex-bridge`](https://github.com/eLyiN/codex-bridge)（MCP）在 CC 会话内同步「问一句 GPT」。当前先用双 DeepSeek 会话跑起来。

## 3 · 风格宪法（所有美术与 UI 验收对照）

方向已由现有素材收敛（GRIS / Journey 气质）：**扁平插画 + 低饱和统一色调 + 雾气遮蔽 + 点状光源 + 一点高饱和的刺**。

- **色板（森林世界，取自设计文档 CSS 变量）**：叶远 `#9EB092` / 叶中 `#809C72` / 叶近 `#688A5C`；枝 `#B06044`；暖光 `#F8D296`；墨 `#2B3642`；背景 `#FAECD4→#E0E0D2`。每个世界一套，MVP 只做森林（青雾）。
- **线条**：扁平剪影为主，描边克制。
- **构图**：雾负责「看不清」，意象不必具象；光源数量有上限。
- **动效 = 程序化**：视差 + 雾漂移 + 粒子 + 6 秒呼吸缩放。运行时零生成成本，永远丝滑。
- **禁止项（硬性）**：❌ 清晰人脸（脸一律背影/剪影，规避恐怖谷与肖像风险）；❌ 纯黑；❌ 血腥直出（一律剪影化 / 红雾）。
- 所有 AI 生成图**先过白名单转译**：人脸→剪影，血→红雾，再进图像模型。

## 4 · 安全铁律（不可协商，写进系统提示词）

这是一个接住脆弱倾诉的产品，安全**不分期，MVP 即全量**。

**倾听者三铁律**：① 不下指令；② 不评判、不诊断、不解读符号；③ 不无边界地肯定。
补充：话要少、慢、留白；在场重于开口；不承诺「我会一直在」这类加深依恋的话。

**五层安全**（详见设计文档 §4）：L1 输入风险检测（微信 msgSecCheck + 关键词层 + LLM 语义分类）→ L2 对话约束 → L3 会话护栏 → L4 角色生命周期 → L5 合规红线。

**危机分级响应**：
- 高危（自伤/自杀/伤人意念）→ 立即停止一切探索性提问，切危机版安放，打破第四面墙给出真实求助资源：全国心理援助热线 **12356**（⚠️ **上线前必须核实当前号码**）。
- 中危（妄想信号）→ 倾听者温和自我拆解（世界观内 reality-check）。
- 复刻创伤梦反复 → 不阻断不点破，转纯复述、加权安放。

**误拦截兜底**：内容安全接口把创伤倾诉判违规时，**绝不弹「内容违规」**；本地仍存入容器，倾听者温柔接住。

**角色生命周期红线**：倾听者上线后任何变更走「世界观内温和过渡」，永不让它突然消失或性情大变。

## 5 · 升级规则（这些必须由 Claude 评审，不许两个便宜 agent 自行拍板）

任何触及以下内容的代码/文案，Coder 与 Reviewer 一致后仍须标 `NEEDS-ARCHITECT` 等 Claude 过：

1. 五层安全架构的任何一层（L1–L5）、危机响应话术与热线逻辑；
2. 数据主权：本地加密、「化雾」真删、服务端不落库对话原文；
3. 倾听者系统提示词（三铁律的落地）；
4. 合规相关：类目表述、AI 生成内容标识、「助眠」等广告法边界表述；
5. 架构级决策（技术栈替换、数据结构、云函数边界）。

## 6 · 编码规范

- **形态**：微信小程序（工具-笔记/效率 或 生活服务类目，**避开医疗-心理咨询**）；企业主体。文案禁用「治疗/疗愈/心理干预/改善睡眠」，用「陪伴/记录/放松」。
- **前端**：原生小程序（Skyline 渲染 + worklet 动画转场 + canvas 2D/WebGL 做雾与粒子）。**不上 Cocos**。
- **开发流**：**先在 Web（HTML/canvas）与 AI 协作快速迭代原型**，验证过的效果再移植小程序 canvas（两套代码几乎同构）。
- **后端**：微信云开发（云函数 + 云数据库），云函数做三件事：对话转发、安全分级、显影任务队列。
- **LLM**：接已备案国内模型 API（对话用中档、安全分类用便宜小模型旁路，互为冗余）。
- **图像**：即梦 / 通义万相 / 混元，选型看参考图风格一致性能力。

## 7 · 每次会话开场自检（agent 照做）

1. 读本文件 + `ROADMAP.md` + 你要做的 `tasks/T-xxx.md`。
2. 确认自己的 owner 角色（coder / reviewer）。
3. 有冲突时以 `verdant-mirage-product-design.html` 为准。
4. 碰到 §5 升级项 → 停下，走升级路径。
