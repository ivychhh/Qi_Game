# T-001 · 风格实验室 `prototype/style-lab.html`

**负责**：Coder（实现）→ Reviewer（评审）  **依赖**：无  **分支**：`feature/T-001-style-lab`

---

## 为什么做（背景）

设计文档 §6.2 点名的**第一个交付物**。风格是「模糊的感觉」，无法用形容词对齐，只能用**可调的实物**对齐：两位主创对着同一块屏幕拉滑杆，拉到「就是这个」的瞬间点复制，参数就是结论。它的产出直接喂给 T-002「冻结风格宪法」。

先读 `AGENTS.md` §3（风格宪法）与 §6（编码规范）再动手。

## 做什么（范围）

一个**单文件、零构建、纯前端**的 `prototype/style-lab.html`：Canvas 实时渲染森林舞台，所有风格参数可实时拉动，满意后一键导出参数。

### 画面（Canvas 实时预览）
- 分层森林剪影（远/中/近 3–4 层）+ 视差
- 2–3 层雾漂移
- 萤火粒子（正弦漂移、发光）
- 全场景呼吸缩放
- 默认竖屏手机比例（对齐小程序），可切换全屏预览

### 控制面板（滑杆 / 开关）
| 组 | 参数 |
|---|---|
| 雾 | 雾量(不透明度)、漂移速度、层数 |
| 色调 | 基础色板（森林预设为默认起点）、整体色相偏移、饱和度 |
| 光点 | 数量、大小、辉光强度、漂移范围、速度 |
| 律动 | 呼吸幅度(%)、周期(秒) |
| 视差 | 层数、鼠标/陀螺仪强度 |
| 线条 | 描边开关 + 粗细 |

### 导出
- 「复制参数」按钮 → 复制一份 **JSON + `:root` CSS 变量块**到剪贴板（可直接粘进风格宪法与后续场景代码）
- 预设的保存/载入用**内存 + 导出/导入 JSON 文本**（不依赖后端）

## 技术约束
- 纯 HTML/CSS/canvas，无构建步骤；如需库只用 `cdnjs.cloudflare.com`。
- 起点色板用 `AGENTS.md` §3 森林 hex。
- **遵守风格宪法禁止项**：不出现清晰人脸、不用纯黑、无血腥直出。
- 代码结构要便于日后移植小程序 canvas（渲染逻辑与 DOM 控件解耦，参数集中在一个 config 对象）。
- 尊重 `prefers-reduced-motion`：开启时停用粒子动画。

## 验收标准（Coder 自测 + Reviewer 逐条过）
1. 双击本地打开即运行，无报错、无需联网构建。
2. 每个滑杆都实时改变画面，无明显卡顿（目标 60fps；中端机 ≥ 30fps）。
3. 「复制参数」产出的 JSON/CSS 可被重新导入并复现同一画面。
4. 竖屏/全屏切换正常；窗口缩放不破图。
5. 默认画面符合风格宪法气质（雾遮蔽、点状光、扁平剪影），无禁止项。
6. `prefers-reduced-motion` 生效时动画降级不崩。
7. 渲染 config 与控件解耦（为 T-017 小程序移植铺路）。
