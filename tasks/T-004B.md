# T-004b · 入口仪式 · 森林之门（主创 2026-07-12 拍板新增）

**负责**：Coder（实现）→ Reviewer（评审）→ 主创终审  **依赖**：T-004 合并后开工  **分支**：`feature/T-004b-gate`

先读 `design/style-constitution.md`。本单来自主创决定：MVP 要有「站在入口前」的仪式感，哪怕只有森林一扇门。

## 为什么做

打开应用的第一眼不该是加载或直接坠入场景，而是**一个停顿**：你站在森林之门前，决定「今晚进去」。这个主动的轻触是整场夜间仪式的第一个动作。

## 做什么

在 `forest-stage.html` 的状态机最上游加 `gate` 幕（`gate → video → fadeIn → scene`）：

1. **画面（纯程序化 canvas，遵守冻结宪法）**：雾中一扇由两侧树木合抱而成的门，门缝透出暖辉光（`--color-glow`），少量萤火在门前游弋，全景随 6s 呼吸明暗；上方标题「晨雾未醒」，下方提示「轻触，推开今夜的门」（衬线、低透明度）。
2. **交互**：轻触门扉（判定区=门体+门缝光，不小于屏宽 40%）→ 门缝光涌出漫过全屏（约 1.2s）→ 交给下游：首次会话接穿林视频，回访会话直接 fadeIn 场景。
3. **每次打开都显示门**（它就是开屏）；视频保持 T-004 的「每会话一次」逻辑不变。
4. **参数化**：门画成独立的 `drawGate(ctx, x, w, palette)` 之类可复用函数，V2 多世界横向排多扇门时不重构。
5. **reduced-motion**：门静止（保留静态辉光），轻触后无涌光动画、直接切换。
6. URL 调试参数沿用：`?entry=1` 强制完整流程（门+视频），`?entry=0` 门后直接进场景。

## 技术约束

- 不改 T-004 已验收的视频/交接逻辑，门只是往状态机上游加一幕；触发下游用 T-004 现有入口函数。
- 门的绘制复用 style-lab 的树木/辉光绘制函数，色值全部取自 `design/style-constitution.md` §2。
- 禁止项照旧：无纯黑、无清晰人脸。

## 验收标准

1. 打开即见门：雾、辉光、萤火、呼吸、标题、提示齐全，气质符合宪法（评审对照 §2–§4）。
2. 轻触门 → 涌光 → 首次进视频、回访直进场景，两条路径都真实驱动验证。
3. 门以外区域轻触无效（不许误触进入）。
4. reduced-motion 降级正常。
5. `drawGate` 参数化，改 x/w 能画出第二扇门（评审代码级确认即可）。
6. 场景/视频原有验收不回归。
7. **主创终审**：气质这关由主创亲眼看过才算过。
