# T-004 · 穿林入口 · 集成到 `prototype/forest-stage.html`

**负责**：Coder（实现）→ Reviewer（评审）  **依赖**：T-003 ✅  **分支**：`feature/T-004-entry`

先读 `AGENTS.md` + `design/style-constitution.md`（冻结宪法），再动手。

## 为什么做

仪式的第一幕是「穿过森林抵达倾听者」。现有成片 `开屏动画5.mp4`（竖屏 9:16，青雾色调，末帧=雾中亮起的空地）作为一次性大动作，播完无缝落到 T-003 的实时森林舞台。

## 做什么

在 `forest-stage.html` 里加入口序列（不新建页面）：

1. **视频入口**：页面打开 → 全屏播放 `../开屏动画5.mp4`（`muted playsinline autoplay`，竖屏模式下与 canvas 同宽高、同居中）。
2. **无缝交接**：视频末帧是亮起的雾中空地 → 交叠淡入实时 canvas 场景（建议：视频快结束时 canvas 已在底层渲染，视频层 1.2–1.6s 淡出）。交接瞬间不许闪黑、不许跳色。
3. **跳过**：播放期间显示「轻触跳过」（复用 style-lab 的样式），点击任意处直接走淡入到场景。
4. **三重兜底**（任一触发直接跳到现有 fadeIn→scene 流程）：
   - autoplay 被浏览器拒绝（`play()` promise reject）；
   - 视频加载失败（`onerror` / 3s 超时）；
   - `prefers-reduced-motion`：**完全不播视频**。
5. **只播一次**：正常流程仅首次进入播放；提供 `?entry=1` URL 参数强制重播（方便评审）。

## 技术约束

- 纯前端、无构建；视频用相对路径 `../开屏动画5.mp4`。
- 不改动 T-003 已验收的场景渲染逻辑；入口只是「盖在上面的一层」。
- mode 状态机显式：`video → fadeIn → scene`，每个转移点都真实可达（吸取 T-003 mode 死锁教训）。
- 已知问题记录（不在本单修）：视频右下角有「豆包AI生成」水印，正式版需换无水印导出或裁切；在任务单标注即可。

## 验收标准

1. 双击打开：视频自动播放（静音）、竖屏对齐、无黑屏无报错条。
2. 视频→canvas 交接无闪黑/跳色（评审逐帧截图核对交接前后各 1s）。
3. 「轻触跳过」全程可用，跳过后交互正常（触摸树洞有反馈）。
4. 三重兜底逐一验证：模拟 autoplay 拒绝、改坏路径、模拟 reduced-motion，均落回 fadeIn→scene 且交互正常。
5. `?entry=1` 强制重播生效。
6. 无宪法禁止项；场景部分与 T-003 验收状态一致（不回归）。

**自测要求**：headless 截图 + 真实驱动一次完整入口流程（含跳过路径），不许只 grep。
